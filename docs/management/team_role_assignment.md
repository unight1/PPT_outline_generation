# 四人小组角色分工定稿（M1）

## 项目与周期
- 项目：PPT 大纲智能生成与内容补全系统。
- 周期：2026-03-30 至 2026-06-14。

## 成员与职责（主责 + 备份）

### A. （组长，后端编排）
- 主责：
  - 项目推进与里程碑把控（M1~M5）。
  - 后端架构、接口契约、流程编排与集成发布。
  - 端到端联调与版本冻结。
- 备份：
  - 验收指标看板审阅与周报汇总。
- 主要交付物：
  - API 接口定义文档、集成联调清单、版本发布记录。

### B. （RAG 检索负责人）
- 主责：
  - 检索策略、数据源筛选、向量检索/重排、证据回填。
  - 深度控制策略（L0/L1/L2）与真实性规则。
- 备份：
  - 检索链路工程化（缓存、重试、超时策略）。
- 主要交付物：
  - RAG 技术方案、检索服务模块、来源可信度规则表。

### C. （前端交互负责人）
- 主责：
  - 需求澄清交互页面、任务状态页、结果展示页。
  - 结果结构展示（章节/页面/要点/证据/备注）。
- 备份：
  - Prompt 输出结构可用性评审、前后端联调支持。
- 主要交付物：
  - 交互原型页面、前端字段定义、联调记录。

### D. （测试评估与文档负责人）
- 主责：
  - 评测方案（代码评测/模型评测/人工评测）。
  - 小批量/大批量测试组织，课程文档与答辩材料。
- 备份：
  - 评测数据记录模块联调、缺陷复测追踪。
- 主要交付物：
  - 指标口径文档、测试报告、课程交付文档包。

## 职责边界与协作规则
- 每人主责模块默认拥有技术决策优先权；跨模块变更需在组会上确认。
- 每个任务必须有：负责人、截止时间、验收标准。
- 所有关键需求变更先更新文档再改代码，避免口头变更失真。
- 关键节点（M1/M3/M5）由 A 组织评审会，B/C/D 提交书面结果。

## 定稿确认
- 生效时间：即日起。
- 变更方式：仅在组会中提案并形成书面更新。

# M1 任务拆分与责任绑定（方案定稿）

## 目标
在 M1 完成「系统架构与方案设计」：冻结 Prompt 策略、RAG 架构、接口边界、评测口径。

## 时间窗口
- 建议窗口：本周内完成。
- 里程碑检查：周中一次中检，周末一次定稿评审。

## 任务清单（负责人 / 截止 / 验收标准）

| 编号 | 任务 | 负责人 | 协作人 | 截止时间 | 验收标准 |
|---|---|---|---|---|---|
| M1-1 | 冻结任务状态流与接口边界（TaskSpec、Task、Generate） | A | C | D+2 | 形成可评审 API 文档，字段完整且前后端一致 |
| M1-2 | 冻结检索方案（数据源、检索层级、真实性规则） | B | A | D+3 | 有书面策略文档，含 L0/L1/L2 触发条件和限流策略 |
| M1-3 | 冻结澄清交互字段（场景/受众/目标/页数/风格/深度） | C | A | D+3 | 前端字段定义表完成，字段与后端契约一一对应 |
| M1-4 | 冻结评测口径（时间/准确率/满意度/对比基线） | D | A/B/C | D+4 | 评测规则文档完成，可用于后续测试执行 |
| M1-5 | 确认基础环境一致性（本地运行、端口、依赖） | A | B/C/D | D+2 | 四位成员可在本机运行前后端 + MySQL/Redis |
| M1-6 | 完成 M1 评审会与行动项归档 | A | B/C/D | D+5 | 形成评审纪要，列出 M2 输入清单 |

> 说明：D 表示分工会议当天。

## 每日检查点
- A：追踪风险、同步阻塞事项。
- B：更新检索方案进度与难点。
- C：更新交互字段变更与联调问题。
- D：更新评测定义与样例集准备进度。

## M1 完成判定
- 文档齐备：接口、检索、交互、评测四份定稿。
- 责任清晰：每份文档均有 owner 与版本。
- 可执行：M2 可以直接进入「核心生成模块开发」。
